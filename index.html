Token & Agent

More tokens = more context & capability, but also more cost. Think phone bill — pay per word, per request.

Claude Models — Which One Should You Use?

⚡

Haiku

Fastest and most lightweight. Great for simple, repetitive tasks where speed matters more than depth. Lower cost per token.

Speed > Accuracy
📊

Sonnet

The balanced everyday model. Good accuracy, reasonable speed. Works well for most coding and document tasks.

Balanced
👑

Opus ★ Recommended

The most powerful and accurate model. Best for complex reasoning, long tasks, nuanced writing — this is what we use for serious work.

Accuracy > Speed
💡 Our recommendation: Use Opus

For finance work, document generation, and any task where correctness matters — Opus is the right choice. The quality difference is very noticeable on complex or multi-step tasks.

The Lesson: A Project That Didn’t Make It

💡

The Idea

Many colleagues were still waiting to get Copilot access. So I built a “Fake Copilot” — an Office.js add-in powered by Claude that would deliver ~80% of Copilot’s features to anyone, for free.

The code was ready. The install package was ready. Everything was in place.

🚫

The Blocker

SAP’s IT policy blocks all three sideload channels for Office add-ins. The package existed but could not be installed on any SAP laptop.

Lesson: Always check company permissions before investing in the build. AI can build anything — but corporate constraints are real walls.

💬 The honest takeaway

Failure is part of building. The instinct was right — identify a gap, solve it with AI. The limitation wasn’t the technology, it was the permission layer. As AI capabilities grow, so will the pressure on companies to open more access.
